# アクセスログ機能 要件定義書

**プロジェクト**: 京都医塾 ポータルサイト（kiweb）  
**対象システム**: teacher-auth  
**作成日**: 2025年3月3日  
**バージョン**: 1.0  

---

## 1. 概要

### 1.1 目的

ポータルサイトへのアクセス履歴を記録し、管理者が閲覧・分析できるようにする。

### 1.2 背景

- 利用状況の把握
- セキュリティ監視・不正アクセス検知
- 障害時の原因調査
- 監査・証跡の確保

### 1.3 スコープ

| 対象 | 内容 |
|------|------|
| 記録対象 | ポータル入口（portal-guard.php）へのアクセス |
| 閲覧対象 | 管理者画面からのログ一覧・検索 |
| 対象外（将来検討） | iframe内ページ単位の記録、API呼び出しの記録 |

---

## 2. 機能要件

### 2.1 ログ記録機能

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-001 | 認証済みユーザーがポータルにアクセスした際にログを記録する | 必須 |
| F-002 | 未認証（ログイン前）のアクセスは記録しない | 必須 |
| F-003 | ログ記録の失敗がポータル表示に影響しない（非同期・エラー時はスキップ可） | 必須 |

### 2.2 記録項目

| 項目 | 必須 | 説明 | 備考 |
|------|------|------|------|
| user_id | ○ | ユーザーID（UUID） | users.id と紐付け |
| page_path | ○ | アクセスしたページパス | 例: kiweb2.html, kiweb2-admin.html |
| request_method | ○ | HTTPメソッド | 通常 GET |
| ip_address | ○ | クライアントIP | RateLimiter::getClientIp() を使用 |
| user_agent | △ | ブラウザ・OS情報 | 任意 |
| referer | △ | 参照元URL | 任意 |
| created_at | ○ | アクセス日時 | 自動設定 |

### 2.3 ログ閲覧機能（管理者画面）

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-010 | 管理者画面に「アクセスログ」メニューを表示する | 必須 |
| F-011 | ログ一覧をテーブル形式で表示する | 必須 |
| F-012 | 日付範囲でフィルタできる | 必須 |
| F-013 | ユーザー名・メールで検索できる | 必須 |
| F-014 | ページパスでフィルタできる | 推奨 |
| F-015 | ページネーションで表示する（1ページ50件程度） | 必須 |
| F-016 | 総件数を表示する | 推奨 |

### 2.4 ログ保持・クリーンアップ

| 要件ID | 要件 | 優先度 |
|--------|------|------|
| F-020 | 90日を超えたログを自動削除する | 必須 |
| F-021 | クリーンアップは bootstrap.php の既存処理に組み込む | 必須 |

---

## 3. 非機能要件

### 3.1 パフォーマンス

| 要件ID | 要件 |
|--------|------|
| NF-001 | ログ記録によるポータル表示の遅延は 100ms 以内とする |
| NF-002 | ログ一覧の初回表示は 3 秒以内とする |

### 3.2 セキュリティ

| 要件ID | 要件 |
|--------|------|
| NF-010 | ログ閲覧は管理者権限（manage_users または view_audit_logs）に限定する |
| NF-011 | ログ記録に機密情報（パスワード、トークン等）を含めない |

### 3.3 可用性

| 要件ID | 要件 |
|--------|------|
| NF-020 | ログ記録の失敗時もポータルは正常に表示する |

## 4. データベース設計

### 4.1 テーブル定義

**テーブル名**: `access_logs`

| カラム名 | 型 | NULL | デフォルト | 説明 |
|----------|-----|------|------------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主キー |
| user_id | CHAR(36) | YES | - | ユーザーID（users.id） |
| page_path | VARCHAR(500) | NO | - | ページパス |
| request_method | VARCHAR(10) | NO | 'GET' | HTTPメソッド |
| ip_address | VARCHAR(45) | YES | - | IPアドレス |
| user_agent | TEXT | YES | - | User-Agent |
| referer | VARCHAR(500) | YES | - | Referer |
| created_at | TIMESTAMP | NO | CURRENT_TIMESTAMP | 作成日時 |

### 4.2 インデックス

| インデックス名 | カラム | 用途 |
|----------------|--------|------|
| PRIMARY | id | 主キー |
| idx_user_id | user_id | ユーザー別検索 |
| idx_created_at | created_at | 日付範囲検索 |
| idx_page_path | page_path(100) | ページ別検索 |

### 4.3 マイグレーション

- ファイル: `teacher-auth/database/migration_add_access_logs.sql`
- 既存DBへの追加のみ（既存テーブル変更なし）

## 5. API仕様

### 5.1 アクセスログ一覧取得

**エンドポイント**: `GET /kiweb/teacher-auth/api/admin/access-logs.php`

**権限**: `manage_users` または `view_audit_logs`

**クエリパラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|------------|-----|------|------|
| page | int | - | ページ番号（1始まり） |
| per_page | int | - | 1ページあたり件数（デフォルト50、最大100） |
| date_from | string | - | 開始日（YYYY-MM-DD） |
| date_to | string | - | 終了日（YYYY-MM-DD） |
| user_search | string | - | ユーザー名・メールの部分一致 |
| page_path | string | - | ページパスの部分一致 |

**レスポンス例**:

```json
{
  "items": [
    {
      "id": 1,
      "created_at": "2025-03-03 14:30:00",
      "user_id": "xxx-xxx-xxx",
      "user_name": "山田太郎",
      "user_email": "yamada@example.com",
      "page_path": "kiweb2.html",
      "request_method": "GET",
      "ip_address": "xxx.xxx.xxx.xxx",
      "user_agent": "Mozilla/5.0..."
    }
  ],
  "total": 123,
  "page": 1,
  "per_page": 50
}
```

**エラー**:

| ステータス | 内容 |
|------------|------|
| 401 | 未認証 |
| 403 | 権限なし |

## 6. 画面仕様

### 6.1 管理者画面 - アクセスログ

**表示場所**: 管理画面サイドバー「ログ」→「アクセスログ」

**レイアウト**:

- 左パネル: フィルタ
- 右パネル: ログ一覧テーブル

**フィルタ項目**:

- 日付範囲（開始日・終了日）
- ユーザー検索（名前・メール）
- ページパス検索
- 検索ボタン

**テーブル列**:

| 列 | 表示内容 | 幅目安 |
|----|----------|--------|
| 日時 | created_at | 160px |
| ユーザー | user_name | 120px |
| メール | user_email | 180px |
| ページ | page_path | 150px |
| IP | ip_address | 120px |
| User-Agent | user_agent（省略表示） | 残り |

**その他**:

- ページネーション（前へ・次へ・ページ番号）
- 総件数表示（例: 「全 123 件」）

### 6.2 権限

- アクセスログ閲覧: `manage_users` 権限を持つユーザー
- 将来的に `view_access_logs` を追加する場合は、その権限でも閲覧可能とする

## 7. プライバシー・個人情報

| 項目 | 対応 |
|------|------|
| IPアドレス | 記録する。必要に応じてマスキング検討 |
| ユーザーID | 記録する（利用状況把握に必要） |
| User-Agent | 記録する（環境把握・異常検知に使用） |
| Referer | 記録する（任意） |
| 保存期間 | 90日（クリーンアップで削除） |
| アクセス制限 | 管理者権限者のみ閲覧可能 |

## 9. テスト観点

| 観点 | 内容 |
|------|------|
| ログ記録 | ポータルアクセス時にログが記録されること |
| ログ未記録 | 未認証時はログが記録されないこと |
| 閲覧権限 | 管理者のみログ一覧が表示されること |
| フィルタ | 日付・ユーザー・ページで正しく絞り込めること |
| ページネーション | 大量データのページ切り替えが正しく動作すること |
| クリーンアップ | 90日以上前のログが削除されること |

## 10. 今後の拡張（検討事項）

| 項目 | 説明 |
|------|------|
| iframe内ページの記録 | 各HTMLページの読み込み時にログAPIを呼ぶ |
| API呼び出しの記録 | 各APIのリクエストを記録 |
| CSV出力 | ログのエクスポート機能 |
| 監査ログとの統合 | audit_logs と access_logs の一覧表示 |
| view_access_logs 権限 | 管理画面全体ではなくログ閲覧のみの権限 |
